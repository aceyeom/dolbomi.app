# 07 · 월간 결산 (Monthly Recap)

> **상태 박스**
> 우선순위: 빌드 순서 6번. MVP의 유일한 오가닉 획득 루프.
> 의존성: AI 파이프라인([03](./03-opportunity-engine.md) §3.4·§3.6, 배치 호출), 스탯/적금/휴가 사다리 데이터.
> 미해결 질문: §7.5 참조.

매월 1일, 카카오톡 공유용으로 만든 카드. **MVP의 유일한 오가닉 획득 루프** — 사용자가 자랑하면 그게 곧 광고다. 그래서 카드는 *자랑스럽되 사생활을 안 흘려야* 한다.

---

## 7.0 약점 / 리스크 (먼저 솔직하게)

1. **공유 안 하면 획득 루프가 0이다.** 유일한 오가닉 채널인데 공유율이 낮으면 성장 엔진이 꺼진다. → 해결: 카드는 **자랑할 만한 숫자**(적금 누적·휴가 사다리·스탯 점프)를 전면에, 부끄러운 데이터(미달·저조)는 절대 노출 안 함(§7.2). 공유는 *성취의 표현*이지 *보고*가 아니게.
2. **개인정보·민감정보 노출.** 카드에 부대·실명·약점이 새면 사용자가 공유를 안 한다(군 보안 의식 높음). → 해결: 카드는 **집계 숫자만**, 부대·실명·민감 자격 0(§7.3, → [10](./10-honesty-notes.md)). 공유 전 미리보기 필수.
3. **제네릭 카피 = 안 자랑함.** "이번 달도 수고!" 같은 톤은 공유 가치 0. → 해결: 카피는 **그 사람의 실제 변수**(N개 퀘스트·+X 숙련도·+XX만 적금)로 채운 템플릿, 변수 빈약하면 *공유 권유 안 함*(§7.1).
4. **휴가 사다리 과약속이 카드로 번짐.** "휴가 +N일"을 단정하면 공유돼서 퍼지는 만큼 신뢰 리스크도 퍼진다. → 해결: 카드의 휴가 표기도 **범위 + "부대별 상이"** 강제(§7.2, → [03](./03-opportunity-engine.md) §3.8).
5. **빈 달(저활동) 처리.** 활동이 거의 없는 달에 카드를 들이밀면 죄책감. → 해결: 임계 미만이면 카드 **생성 안 함** 또는 "다음 달 목표" 톤으로 전환(§7.4).

---

## 7.1 카드 — 무엇을 담나

매월 1일 생성, 카카오톡 공유 최적화:

> *"이번 달: 퀘스트 N개 · 숙련도 +X · 적금 +XX만 · 휴가 사다리 +N일(부대별 상이) · 다음 목표 D-XX."*

**변수 소스(전부 집계, 민감정보 0):**

```jsonc
RecapCard {
  month,
  quests_done,                 // Σ 완료 Quest
  stat_deltas: {craft:+X, ...},// 월초 대비 레벨/XP 증가
  savings_delta,               // 적금 누적 증가(→ 04 money)
  ladder_range: [min,max],     // 휴가 사다리 범위(→ 03 §3.8), 단정 금지
  next_dday                    // 가장 임박한 기회 D-day
}
```

카피는 **같은 AI 파이프라인의 싼 배치 호출**(→ [03](./03-opportunity-engine.md) §3.6)로 생성 — 매일 호출 아님, 월 1회 배치라 비용 무시 가능. 변수가 빈약하면(§7.4) 공유 권유 자체를 띄우지 않는다.

## 7.2 자랑 설계 (공유율 = 성장)

- **전면:** 자랑스러운 숫자만(적금 누적·휴가 범위·가장 큰 스탯 점프).
- **숨김:** 미달·저조·뒤처진 스탯·약점은 카드에 **안 들어감**(§7.0-1).
- **휴가 표기:** 반드시 `{min}~{max}일 · 부대별 상이` — 단정 금지(§7.0-4).
- 디자인: 솔로 레벨링 톤의 트로피 카드, 한 장 이미지로 톡 공유.

## 7.3 개인정보 가드 (군 보안 + 신뢰)

- 카드는 **집계 숫자만**. 부대명·실명·민감 자격·journal 텍스트 **절대 비포함**(→ [10](./10-honesty-notes.md), [03](./03-opportunity-engine.md) §3.9).
- **공유 전 미리보기 필수** — 사용자가 정확히 무엇이 나가는지 보고 누름.
- 기본 비공개; 공유는 항상 명시적 액션.

## 7.4 빈 달 / 저활동 처리

```
if quests_done < THRESH (시드 5):
   카드 생성 안 함 (또는 "다음 달 첫 목표" 격려 톤으로 전환)
```
죄책감 카드 금지(§7.0-5). 자랑할 게 없으면 자랑을 강요하지 않는다.

## 7.5 미해결 질문 (Phase 2/실데이터)

- 카카오톡 공유 카드 포맷(이미지 규격·딥링크 백 채널).
- 공유율 실측 + 어떤 변수가 공유를 유발하는지(적금 vs 휴가 vs 스탯).
- 저활동 임계(THRESH=5)와 격려 전환 톤 A/B.
- 카드 → 신규 설치 어트리뷰션(오가닉 루프 효과 측정).
