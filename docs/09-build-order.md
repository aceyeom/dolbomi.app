# 09 · 빌드 순서 (Build Order)

> **상태 박스**
> 우선순위: 실행 순서의 헌법.
> 의존성: 모든 파트.
> 미해결 질문(Phase 2 후보): 각 단계의 "완료 정의(DoD)"와 검증 게이트.

MVP 내부 구현 순서:

1. **코어 루프 + 스탯 + Gap + D-day/스트릭 + 위젯** — 일일 리텐션을 증명하는 껍데기. (→ [01](./01-core-loop.md), [02](./02-stats.md))
2. **퀘스트 경로 생성기** (→ [03](./03-opportunity-engine.md) §3.3) 를 **시드된 3–5개 기회**(적금, 정보처리, TOEIC, 한국사, 창업대회)에 대해. *진짜 병사로 테스트. 경로가 제네릭하면 다른 무엇보다 먼저 여기서 멈추고 고친다 — 이게 전부의 베팅이다.*
3. **기회 바 + 휴가 사다리** 를 생성기에 배선. (→ [03](./03-opportunity-engine.md) §3.2, §3.4)
4. **혜택 허브** (셀프 필터) + "퀘스트로 만들기" 핸드오프. (→ [05](./05-benefits-hub.md))
5. **리퍼럴 엔진** (2단계 완료) 를 자격증/어학/금융 퀘스트에. (→ [06](./06-referral-engine.md))
6. **월간 결산.** (→ [07](./07-monthly-recap.md))
7. **전체 Part 04 카탈로그** 시딩, 각 항목에 `source_url` + 검증일. (→ [04](./04-catalog/README.md))
