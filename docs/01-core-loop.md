# 01 · 코어 루프 (The Spine)

> **상태 박스**
> 우선순위: 빌드 순서 1번 (→ [09](./09-build-order.md)). 일일 리텐션을 증명하는 껍데기.
> 의존성: 스탯([02](./02-stats.md)), 기회 엔진([03](./03-opportunity-engine.md))의 `next_3` 후보, AI 파이프라인([03](./03-opportunity-engine.md) §3.3).
> 미해결 질문(Phase 2 후보): 체크인 스킵/미접속일의 처리, 위젯 갱신 주기, 명예 탭의 어뷰징 방지.

코어 루프는 매일 반복되는 **체크인 → 오늘 밤의 3 → 완료 → 보상 클러스터 → 한 줄** 의 5단계다.

---

## 1.1 60초 체크인

잠금 해제 시 두 번 탭: **기분(mood, 5점)** + **에너지(energy, low / mid / high)**. 일일 행 `{date, mood, energy}` 로 저장.

이건 분석용 장식이 아니다 — **오늘 밤 퀘스트의 크기를 정하는 입력값**이다. 선택적 한 줄 회고(자유 텍스트)는 AI 컨텍스트(→ [03](./03-opportunity-engine.md) §3.3)로 들어가지만 **절대 필수가 아니다.**

## 1.2 오늘 밤의 3

오늘 밤을 위해 3개의 퀘스트가 떠오른다. **새로 생성하는 게 아니라 선택(selected)** — (비용 통제 → [03](./03-opportunity-engine.md) §3.3) 다음 두 집합의 합집합에서 고른다:
- 사용자의 활성 기회 경로들의 `next` 서브퀘스트
- 상시 습관 퀘스트(standing habit quests)

**오늘 밤 에너지에 따른 선택 규칙:**

| 에너지 | 퀘스트 사이징 |
|---|---|
| low | 3개 모두 **S** (≤5분) 마이크로 퀘스트 — "앱에서 5분", "한 줄 회고", "물 1L" |
| mid | S/M 혼합 |
| high | **L** 1개 허용, 뒤처진 기회를 에스컬레이션 |

앱은 **절대** 지친 병사에게 2시간짜리 퀘스트를 주지 않는다. 이 한 가지 규칙이 핵심 이탈 방지 장치다.

## 1.3 완료 + 애니메이션

한 탭 = 완료 (명예 탭, → [06](./06-referral-engine.md) §6.3). 완료 시 솔로 레벨링 스타일의 레벨업 애니메이션 — 벌어낸 도파민. 선택적 자랑 사진(기본 비공개).

## 1.4 보상 클러스터 (완료 시 움직이는 것들)

- 해당 **스탯 바**가 찬다 (XP, → [02](./02-stats.md) §2.2)
- **기회 진행바**가 전진 + 페이싱 마커 갱신 (→ [03](./03-opportunity-engine.md) §3.2)
- **스트릭**이 올라간다 (월 1회 프리즈로 용서)
- 위젯의 **D-day** 와 **적금** 숫자 갱신

## 1.5 한 줄

루프 후 단 한 줄의 비-제네릭 문장. 그날의 컨텍스트에서 싸게 생성(AI 파이프라인에 올라탐, → [03](./03-opportunity-engine.md) §3.3).
예: *"3일 연속 숙련도 채웠다. 정보처리 필기까지 D-9, 페이스 좋아."*
구체적이고, 진짜 기회에 묶여 있고, 절대 공허한 칭찬이 아니다.
