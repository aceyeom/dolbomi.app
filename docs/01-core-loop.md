# 01 · 코어 루프 (The Spine)

> **상태 박스**
> 우선순위: 빌드 순서 1번 (→ [09](./09-build-order.md)). 일일 리텐션을 증명하는 껍데기.
> 의존성: 스탯([02](./02-stats.md)), 기회 엔진([03](./03-opportunity-engine.md))의 `next_3` 후보, AI 파이프라인([03](./03-opportunity-engine.md) §3.3).
> 미해결 질문: §1.8 참조.

코어 루프는 매일 반복되는 **체크인 → 오늘 밤의 3 → 완료 → 보상 클러스터 → 한 줄** 의 5단계다. 이 루프 하나가 *일일 리텐션* 을 증명한다 — 기회 엔진([03](./03-opportunity-engine.md))이 "베팅"이라면, 코어 루프는 그 베팅이 매일 한 번 사용자를 데려오는지 검증하는 **최소 껍데기**다.

---

## 1.0 약점 / 리스크 (먼저 솔직하게)

1. **루프가 너무 가벼우면 습관이 안 생긴다.** 60초 체크인 + 한 탭 완료는 마찰이 거의 0이라 좋지만, *너무* 싸면 사용자가 "오늘 안 해도 그만"으로 흘린다. → 해결: 루프의 무게중심은 **보상 클러스터(§1.4)의 가시적 전진**(스탯 바·D-day·적금 숫자가 *동시에* 움직임)에 둔다. 마찰은 낮추되 *결과의 가시성*은 높인다.
2. **미접속일(skip)이 스트릭을 깨면 이탈한다.** 군 생활은 훈련·당직·통제로 폰을 못 보는 날이 구조적으로 존재. 스트릭이 그걸 처벌하면 가장 성실한 사용자가 가장 크게 다친다. → 해결: 월 1회 자동 프리즈 + **통제일(훈련/당직) 셀프 선언 시 스트릭 보존**(§1.6). 단 어뷰징 방지 상한.
3. **체크인이 "분석용 장식"으로 전락.** mood/energy를 받아만 두고 그날 퀘스트 크기에 안 쓰면 사용자는 곧 무성의하게 찍는다. → 해결: energy는 **그 즉시 §1.2 사이징의 하드 입력**이 되어야 하고, 사용자는 "낮음 찍으면 진짜 쉬운 것만 나온다"를 1일차에 체감해야 한다.
4. **명예 탭(자기보고) 어뷰징.** 한 탭 완료는 검증이 없어 XP 인플레가 가능. → 해결: 명예 탭 XP는 그대로 두되, *진짜 stakes*(휴가 사다리·verified 배지·리퍼럴 보너스)는 검증 경로(→ [06](./06-referral-engine.md) §6.3)에만 묶는다. 자기보고는 동기부여용, 검증보고는 보상용.
5. **"한 줄"이 제네릭하면 신뢰 즉사.** "오늘도 수고했어요" 같은 공허한 칭찬은 이 코호트가 가장 싫어하는 톤. → 해결: 한 줄은 **그날의 실제 기회·숫자에 묶인 템플릿**(§1.5), 변수 없으면 생성 안 함(빈 줄이 가짜 칭찬보다 낫다).
6. **위젯이 안 죽으면 앱을 안 연다.** 위젯이 D-day·적금을 다 보여주면 사용자가 앱을 열 이유가 약해짐(앱이 자기 위젯에 식인당함). → 해결: 위젯은 *상태*(D-day, 적금, 오늘 완료 여부)만, *행동*(오늘 밤의 3 수행·보상 애니메이션)은 앱에서만. 위젯은 미끼지 식사가 아니다.

---

## 1.1 60초 체크인

잠금 해제 후 두 번 탭: **기분(mood, 1–5)** + **에너지(energy: low / mid / high)**. 일일 행으로 저장:

```jsonc
DailyCheckin { user_id, date, mood:1..5, energy:"low"|"mid"|"high", journal_line? }
```

이건 분석용 장식이 아니다 — **오늘 밤 퀘스트의 크기를 정하는 입력값**이다(§1.2). 선택적 한 줄 회고(자유 텍스트)는 AI 컨텍스트(→ [03](./03-opportunity-engine.md) §3.3)로 들어가지만 **절대 필수가 아니다.**

**규약**
- 하루 1행. 같은 날 재체크인은 **덮어쓰기**(최신 energy가 그날 사이징 지배).
- `journal_line` 은 옵셔널·민감정보 가능 → **텔레메트리 제외**(→ [10](./10-honesty-notes.md), [03](./03-opportunity-engine.md) §3.9).
- 미체크인 날: §1.2가 **`energy="low"` 로 폴백**(지친 날 가정이 안전 — 절대 L을 안 띄움).

## 1.2 오늘 밤의 3

오늘 밤을 위해 3개의 퀘스트가 떠오른다. **새로 생성하는 게 아니라 선택(selected)** — 비용 통제(→ [03](./03-opportunity-engine.md) §3.6)를 위해 LLM 호출 없이 캐시 풀에서 고른다. 선택 로직 전체는 [03](./03-opportunity-engine.md) §3.3 `pickTonight()` 가 소유; 여기서는 **사용자가 보는 계약**만 명시한다.

후보 풀 = (활성 기회 경로들의 `next_3` 서브퀘스트) ∪ (상시 습관 퀘스트).

**에너지 → 사이즈 필터 (불변식):**

| 에너지 | 허용 사이즈 | 예시 |
|---|---|---|
| low | **S 만** (≤5분) | "앱에서 5분", "한 줄 회고", "물 1L" |
| mid | S, M | S 1–2개 + M 1개 |
| high | S, M, **L** | 뒤처진 🔴 기회에서 L 1개 에스컬레이션 |

> **핵심 불변식:** low 에너지 밤에는 **절대** M/L이 나오지 않는다. 이 한 규칙이 핵심 이탈 방지 장치다. 앱은 지친 병사에게 2시간짜리 퀘스트를 주지 않는다.

**엣지 케이스**
- 풀이 3개 미만 → 상시 습관으로 채움(**절대 빈 밤 없음**, → [03](./03-opportunity-engine.md) §3.3).
- 모든 활성 기회가 high-에너지 L뿐인데 energy=low → 그 기회들의 S 습관으로 대체, 사용자에겐 "오늘은 가볍게" 카피.
- 같은 기회에서 최대 2개(다양성), 자세한 점수식은 [03](./03-opportunity-engine.md) §3.3.

## 1.3 완료 + 애니메이션

한 탭 = 완료. **명예 탭(self-report)** 이 기본이고 마찰 0(→ [06](./06-referral-engine.md) §6.3). 완료 트랜잭션:

```
onComplete(quest):
  quest.status = "done"; quest.completed_at = now
  awardXP(quest.stat, quest.xp)              // → 02 §2.2 (호스트가 size로 xp 결정)
  if quest.source.opportunity_id:
     advancePath(quest)                       // → 03 §3.2 fill 재계산
  bumpStreak(user)                            // → §1.6
  playLevelUpAnim(quest.stat)                 // 솔로 레벨링 스타일
```

레벨업 애니메이션 = *벌어낸 도파민*. 선택적 자랑 사진(기본 **비공개**). 완료는 **로컬에서 즉시 낙관적 반영**(서버 왕복 대기 없음), 동기화 실패 시 큐잉.

## 1.4 보상 클러스터 (완료 시 동시에 움직이는 것들)

완료 한 번에 **여러 게이지가 동시에** 전진하는 게 이 앱의 도파민 설계의 핵심이다(§1.0-1). 한 탭 → :

- 해당 **스탯 바**가 찬다 (XP, → [02](./02-stats.md) §2.2)
- **기회 진행바**가 전진 + 페이싱 마커 갱신 (→ [03](./03-opportunity-engine.md) §3.2)
- **스트릭**이 올라간다 (§1.6, 월 1회 프리즈로 용서)
- 위젯의 **D-day** 와 **적금** 숫자 갱신 (§1.7)
- 해당되면 **휴가 사다리**(→ [03](./03-opportunity-engine.md) §3.8) 진척

> 설계 의도: 한 번의 작은 행동이 *여섯 방향으로* 가시적 결과를 낸다. 마찰은 낮고(§1.0-1) 결과 가시성은 높다.

## 1.5 한 줄

루프 후 단 한 줄의 비-제네릭 문장. 그날 컨텍스트에서 싸게 생성(배치/템플릿, → [03](./03-opportunity-engine.md) §3.6).

예: *"3일 연속 숙련도 채웠다. 정보처리 필기까지 D-9, 페이스 좋아."*

**생성 규칙 (제네릭 방지):**
- 반드시 **실제 변수**(연속일수·기회 제목·D-day·적금액·스탯 증가분) 중 ≥1개를 포함.
- 변수가 하나도 없으면(예: 첫날, 활성 기회 없음) **생성하지 않음** — 빈 줄이 "오늘도 수고했어요"보다 낫다.
- 톤: 전우의 한 마디. 칭찬 아님, 코치 아님, **상태 보고 + 다음 한 발**.

## 1.6 스트릭 + 프리즈 + 통제일 (군 현실 보정)

`Streak { user_id, count, last_completed, freezes_left }` (→ [08](./08-data-models.md)).

- **카운트:** 그날 ≥1 퀘스트 완료 시 +1.
- **자동 프리즈:** 월 1회. 미완료 하루를 자동 용서(스트릭 유지, 카운트 정지).
- **통제일 보존:** 사용자가 그날을 *통제(훈련/당직/통제)* 로 셀프 선언 → 스트릭 **보존**(끊기지 않음). 군 생활 구조상 폰 불가일을 처벌하지 않기 위함(§1.0-2).
- **어뷰징 가드:** 통제일 선언 **≤ N일/월**(시드 N=8, Phase 2 튜닝). 초과분은 일반 미접속 처리. 프리즈/통제일은 *카운트를 올리지 않음*(완료만 카운트↑) — 가짜 스트릭 인플레 방지.

## 1.7 위젯 (홈 화면)

위젯은 **상태만, 행동은 앱에서**(§1.0-6).

| 위젯 표시 | 출처 | 갱신 |
|---|---|---|
| 가장 임박한 **D-day** | 활성 기회 `schedule` | 완료/날짜변경 시 |
| **적금** 누적 추정 | 적금 기회 진척(→ [04](./04-catalog/money.md)) | 월 납입/완료 시 |
| 오늘 **완료 여부** 점 | 오늘 Quest status | 완료 시 즉시 |

위젯 탭 → 앱의 오늘 밤의 3로 딥링크. 위젯에서는 **완료 불가**(보상 클러스터 애니메이션을 앱에 묶어 앱 오픈을 유지).

## 1.8 미해결 질문 (Phase 2/실데이터)

- 체크인 스킵/미접속일의 정확한 폴백 카피 톤(죄책감 vs 격려 A/B).
- 위젯 갱신 주기와 배터리/OS 제약(iOS 타임라인 vs Android).
- 통제일 셀프 선언 상한(N=8) 어뷰징 실측.
- 명예 탭 XP 인플레가 The Gap(→ [02](./02-stats.md) §2.3) 동기를 희석하는지.
- "한 줄" 생성 빈도(매일 vs 이벤트 있는 날만)와 배치 비용.
