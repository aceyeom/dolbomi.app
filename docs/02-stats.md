# 02 · 스탯 시스템 (Stats)

> **상태 박스**
> 우선순위: 빌드 순서 1번 묶음. 코어 루프와 함께.
> 의존성: 퀘스트의 `xp`/`stat`([03](./03-opportunity-engine.md), [08](./08-data-models.md)).
> 미해결 질문: §2.6 참조.

여섯 개의 스탯이 *현실 역량* 을 게임 게이지로 매핑한다. 스탯은 단순한 점수가 아니라 **The Gap(§2.3)** 을 통해 "전역까지 남은 거리"를 숫자로 만드는 장치다.

---

## 2.0 약점 / 리스크 (먼저 솔직하게)

1. **레벨 곡선이 틀리면 동기 곡선이 틀린다.** 초반이 너무 느리면 1주차 이탈, 후반이 너무 빠르면 트로피 가치가 증발. → 해결: `floor(sqrt(totalXP/100))` 의 완만한 곡선 + 실데이터 튜닝(§2.6). 초반 레벨은 빠르게, 후반은 희소하게.
2. **담력(Edge) XP 희소성 어뷰징.** "회피해온 것"에서만 나오는 희소 XP가 트로피의 핵심인데, 사용자가 일부러 방치했다가 막판에 쳐서 XP를 긁으면 의미가 깨진다. → 해결: 담력 XP는 *회피 플래그가 선 기회를 실제로 전진*시킬 때만, 그리고 **방치 자체엔 보상 없음**(§2.4). 어뷰징 패턴은 텔레메트리로 감시.
3. **The Gap이 불안만 주고 행동을 안 만든다.** 고스트 바(목표)가 라이브 바보다 한참 앞서 있으면 *압도*되어 포기. → 해결: Gap은 항상 **가장 가까운 한 발**(다음 서브퀘스트)로 링크되어야 하고, 거리감은 "막막함"이 아니라 "다음 칸"으로 번역(§2.3).
4. **목표 재설정 악용.** The Gap 목표를 낮추면 즉시 거리가 줄어 가짜 진척감. → 해결: 목표 하향은 허용하되 **로그를 남기고**(자기기만 방지), 하향은 The Gap 진척률에 *소급 반영 안 함*(§2.3). Phase 2에서 재설정 빈도 제한 검토.
5. **스탯 6개가 균등 노출되면 약점 스탯이 영영 안 큰다.** 사용자는 잘하는 것만 함. → 해결: 야간 픽 점수식이 **가장 뒤처진 스탯(The Gap)에 가중**(→ [03](./03-opportunity-engine.md) §3.3), 스탯 시스템은 그 "뒤처짐"을 *가시적*으로 만들 책임(§2.3).

## 2.1 여섯 개의 스탯 (사회 매핑)

| 인앱 이름 | 현실 역량 | 1차 기회 피드 (→ [04](./04-catalog/README.md)) |
|---|---|---|
| 전투력 **Body** | 체력, 건강 | 체력검정, 사격, 헌혈 |
| 정신력 **Mind** | 회복탄력성, 집중, 독서 | 다독왕, 정신전력 공모전, 회고 |
| 자산력 **Money** | 저축, 금융 리터러시 | 적금, 자기개발비, ISA/청약 |
| 숙련도 **Craft** | 자격증, 취업 스킬 | 국가기술자격, TOEIC, 학점취득, 군 e-러닝 |
| 지휘력 **People** | 리더십, 커뮤니케이션 | 분대장, 멘토링 |
| 담력 **Edge** | 용기, 주도성 | 창업경진대회, 공모전, 회피해온 그것 |

각 스탯은 `Stat { user_id, key, xp, level }` (→ [08](./08-data-models.md)). `key ∈ {body,mind,money,craft,people,edge}` — **코드 식별자는 영어 고정**, 표시명만 한국어.

## 2.2 레벨링 수학 (단순하게)

**XP 부여 — 호스트가 결정, LLM 아님:**

```
xp(size) = { S:10, M:25, L:60 }
기회 서브퀘스트 보너스: × 1.5   (현실 stakes가 걸린 퀘스트)
습관 퀘스트: 보너스 없음 (×1.0)
```

> LLM이 출력한 `xp` 값은 **버린다.** 호스트가 `size` 로 재계산(→ [03](./03-opportunity-engine.md) §3.4 검증/리페어). 환각 인플레 차단.

**레벨 곡선:**

```
level(totalXP) = floor( sqrt(totalXP / 100) )
```

| 레벨 | 필요 누적 XP | 칸 사이 간격 |
|---|---|---|
| 1 | 100 | — |
| 2 | 400 | 300 |
| 3 | 900 | 500 |
| 5 | 2,500 | (가속) |
| 10 | 10,000 | (후반=자랑거리) |

초반 레벨은 빠르게 차서 1주차 동기를 주고, 후반 레벨은 희소해져 **스크린샷 트로피**가 된다.

**바 fill:** 현재 레벨 시작 XP ~ 다음 레벨 XP 사이 선형 보간. 레벨업 시 §1.3 애니메이션 트리거.

## 2.3 The Gap (목표 오버레이) — 거의 공짜, 높은 임팩트

온보딩에서 병사가 스탯별 *전역 목표치* 를 설정하고 구체적 목표를 명명한다("정보처리기사 + TOEIC 800 + 적금 만기"). 각 라이브 바 뒤에 **고스트 처리된 두 번째 바**로 렌더.

```
gap_pct(stat)   = clamp( live_level / target_level , 0, 1 )
distance(stat)  = target_level - live_level          // 남은 칸
```

줄어드는 거리가 *뒤처지는 불안* 을 숫자로 바꾼다. **단, 막막함이 되지 않도록:**
- 각 Gap은 **그 스탯의 다음 서브퀘스트**로 직접 링크(다음 한 발, §2.0-3).
- *가장 큰 Gap* = 야간 픽이 우선 채우는 스탯(→ [03](./03-opportunity-engine.md) §3.3 "The Gap 좁히기" 점수).

**목표 재설정:** 하향 허용하되 **변경 로그 기록**, 진척률에 소급 반영 안 함(§2.0-4). 상향은 자유.

빌드 비용 ≈ 이미 그려진 바에 스타일링 한 번 + 고스트 레이어.

## 2.4 담력(Edge) — 희소 XP의 메커닉

담력은 다른 스탯과 부여 규칙이 다르다.

- **회피 플래그:** 어떤 기회 경로가 **7일 이상 무활동** → 그 기회에 `avoided=true` (→ [03](./03-opportunity-engine.md) §3.1 lifecycle, §3.3 `isEdgeAvoided`).
- **희소 XP:** 회피 플래그가 선 기회의 서브퀘스트를 *실제로 완료* 할 때만 담력 XP(+보너스). 즉 "미뤄둔 무서운 것을 드디어 함"에만 보상.
- **방치엔 보상 0:** 방치 자체는 아무것도 안 줌(어뷰징 방지, §2.0-2). 담력은 *행동* 의 대가지 *대기* 의 대가가 아니다.

그래서 담력 바가 가장 채우기 어렵고, 가장 자랑스러운 트로피가 된다.

## 2.5 스탯 ↔ 다른 파트 연결

- **루프:** 완료 → `awardXP(stat, xp)` (→ [01](./01-core-loop.md) §1.4).
- **엔진:** 서브퀘스트가 `stat` 을 들고 옴; 야간 픽이 뒤처진 스탯 가중(→ [03](./03-opportunity-engine.md) §3.3).
- **카탈로그:** 각 `Opportunity.stat` 이 1차 스탯 피드(→ [04](./04-catalog/README.md)).
- **결산:** 월간 스탯 증가분이 공유 카드 변수(→ [07](./07-monthly-recap.md)).

## 2.6 미해결 질문 (Phase 2/실데이터)

- 레벨 곡선 계수(`/100`)·보너스 배율(×1.5)의 실데이터 밸런싱.
- 담력 회피 플래그 임계(7일)와 어뷰징 패턴 탐지.
- The Gap 목표 재설정 빈도 제한 여부·소급 정책.
- 스탯 6개가 균등하게 자라는지(약점 스탯 방치율) 텔레메트리.
- 습관 퀘스트 보너스 부재가 습관 형성 동기를 꺾는지.
