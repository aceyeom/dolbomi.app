# 06 · 리퍼럴 엔진 (Referral Engine) — MVP 1차 매출

> **상태 박스**
> 우선순위: 빌드 순서 5번 (자격증/어학/금융 퀘스트 위에).
> 의존성: 기회 엔진([03](./03-opportunity-engine.md))의 서비스 점프, 카탈로그([04](./04-catalog/README.md))의 `partner_service_id`, 데이터 모델([08](./08-data-models.md)).
> 미해결 질문: §6.6 참조.

리퍼럴 엔진은 MVP의 **유일한 1차 매출원**이자, "노력은 검증할 수 없다"는 근본 문제를 우회하는 장치다. 핵심 통찰: 우리는 *노력* 이 아니라 *인게이지먼트(설치/가입)* 를 검증한다 — 그게 체크 가능하고, 동시에 돈을 주는 그 행위다.

---

## 6.0 약점 / 리스크 (먼저 솔직하게)

1. **이 코호트는 광고 레이더가 예리하다.** "결국 영업이네" 한 번 들키면 떠난다. → 해결: **정부 돈 먼저**(자기개발비·무료검정), 리퍼럴은 자기 돈 경로에만, 투명 라벨링(§6.5). 매출 경로는 *항상 선택*, 절대 페이월 아님.
2. **어트리뷰션 없는 파트너 → 검증 불가 → 어뷰징.** postback 없는 서비스는 스크린샷 폴백을 쓰는데, 스샷은 위조 가능. → 해결: 스샷 폴백은 **명예 탭과 동급 보상(verified 아님)** 으로만, *진짜 보너스*는 postback 검증에만(§6.3). 위조해도 얻는 게 명예 탭 XP뿐이라 어뷰징 유인 제거.
3. **단위경제가 인원수에 묶이면 시즌 비즈니스에서 깨진다.** 입대/전역은 코호트성 — 인원이 출렁인다. → 해결: 매출은 *인원수*가 아니라 *인게이지먼트 이벤트*에 스케일(§6.4). 같은 파이프가 검증과 청구를 동시에.
4. **나쁜 파트너 하나가 신뢰 전체를 오염.** 사기성·고압 영업 서비스를 리퍼하면 앱 전체 신뢰 붕괴. → 해결: 파트너 온보딩 기준(§6.5-1) + "진짜 도움" 게이트. 정부 대체재가 있으면 그걸 우선.
5. **postback 명세 불일치.** 파트너마다 어트리뷰션 방식(CPI/CPA/rev-share)·콜백 포맷이 달라 검증 누락·오청구 위험. → 해결: 표준 `PartnerService` 스키마 + idempotent postback 처리(§6.3, Phase 2 명세 확정).

---

## 6.1 인사이트 ("노력 검증 불가" 문제도 해결)

우리는 *노력* 을 검증하지 않는다. 제휴 서비스로 라우팅하고 *인게이지먼트(설치/가입)* 를 검증한다 — 이건 체크 가능하고(어필리에이트 어트리뷰션) 우리에게 돈을 주는 그 행위다. 퀘스트는 저커밋 **행동**이지 **보고**가 아니다.

> 한 문장 요약: **"공부했다"는 못 믿어도, "이 앱을 깔았다"는 믿을 수 있다 — 그리고 그게 매출이다.**

## 6.2 퀘스트 → 서비스 맵

| 퀘스트 도메인 | 파트너 타입 | 매출 이벤트 |
|---|---|---|
| 어학(TOEIC/OPIc) | 어학 인강 앱 | 설치 / 구독 |
| 자격증 | 자격증 플랫폼 | 강좌 구매(종종 자기개발비 지원) |
| 금융/적금 | 은행, 증권사 | 계좌 개설(CPA) |
| 학습/학점 | 강좌 플랫폼 | 수강 등록 |
| 취업(D-90) | 커리어 플랫폼 | 가입 |

매핑은 카탈로그 `Opportunity.partner_service_id` → `PartnerService`(→ [08](./08-data-models.md))로 연결. 서브퀘스트가 `service_link` 를 들면 §6.3 Tier 2 경로 활성.

## 6.3 2단계 완료

```
Tier 1 (명예 탭)   : 한 탭 "완료"      → 퀘스트 포인트 + 스탯 XP        (검증 없음)
Tier 2 (Verified) : 우리 링크로 도달  → postback ✓ → 보너스 XP + 배지  (검증 있음)
스샷 폴백          : postback 없는 곳  → Tier 1 동급 보상 (verified 아님)
```

- **Tier 1 — 명예 탭 (기본, 마찰 0):** "30분 공부" → 한 탭 "완료." 퀘스트 포인트 + 스탯 XP. **항상 가능; 아무도 페이월 안 됨**(→ [01](./01-core-loop.md) §1.3).
- **Tier 2 — Verified (선택, 더 큰 보상):** 우리 딥링크로 서비스 도달 → 어필리에이트 **postback** 이 퀘스트를 자동 ✓ verified 처리 → 보너스 XP + **verified 배지**(미래 Receipt로 이어짐). 딥링크는 `PartnerService.deep_link_template` 로 생성, 2단계 완료(딥링크 → 복귀)로 흐름 단절 없음.
- **스크린샷 폴백:** 어트리뷰션 없는 곳은 스샷 업로드. **보상은 Tier 1 동급(verified 아님)** — 위조 유인 제거(§6.0-2).
- **넛지:** verified 가 보상↑ + 진짜 신뢰성을 주므로, 강압 없이 사용자가 스스로 매출 경로를 선택.

**Postback 처리(서버):** idempotent — 같은 `(user, service, event)` 중복 콜백은 1회만 반영. 검증 성공 시 `Quest.status = "verified"`, 보너스 XP 부여, 배지 발급.

## 6.4 경제학

**퀘스트를 확정하는 어트리뷰션 postback = 파트너에게 청구하는 그 이벤트** — 한 파이프, 두 출력(검증 + 청구). 모델: CPI / CPA / 구독 rev-share.

```
매출 ∝ Σ (verified 인게이지먼트 이벤트)        // 인원수 아님
```

매출은 인원수가 아니라 인게이지먼트에 스케일 — **시즌 코호트 비즈니스에 fit**(입대/전역이 출렁여도 이벤트 단위로 안정, §6.0-3).

## 6.5 윤리 가드레일 (이 코호트는 레이더가 예리하다)

1. **진짜 도움 되는 서비스만 리퍼** — 파트너 온보딩 게이트(평판·해지 용이성·정부 대체재 유무 점검).
2. **정부 돈 먼저**(자기개발비 / 무료검정) — 병사가 자기 돈이 아니라 군대 돈을 쓰게. 정부 대체재가 있으면 리퍼럴보다 위에.
3. **투명한 제휴 링크 라벨링** — "제휴" 표시, 숨기지 않음.
4. **핵심 약속은 무료 유지** — 리퍼럴을 한 번도 안 탭한 병사도 자기 시간의 완전한 증명을 받는다.

> 이 4개는 협상 불가(→ [10](./10-honesty-notes.md)). 매출 최적화가 이걸 침범하면 매출을 포기한다 — 신뢰가 자산이고, 이 코호트에서 신뢰는 한 번 잃으면 끝.

## 6.6 미해결 질문 (Phase 2/실데이터)

- attribution postback 표준 명세(CPI/CPA/rev-share별 콜백 포맷·idempotency 키).
- 스크린샷 폴백 어뷰징 실측·OCR 검증 도입 여부.
- 파트너 온보딩 기준의 구체 체크리스트.
- verified 배지 → 미래 "Receipt"(전역 이력서) 연결 설계.
- 정부 대체재 우선 정렬이 매출에 주는 영향(수용 가능한 trade-off 범위).
