# 06 · 리퍼럴 엔진 (Referral Engine) — MVP 1차 매출

> **상태 박스**
> 우선순위: 빌드 순서 5번 (자격증/어학/금융 퀘스트 위에).
> 의존성: 기회 엔진([03](./03-opportunity-engine.md))의 서비스 점프, 카탈로그([04](./04-catalog/README.md))의 `partner_service_id`, 데이터 모델([08](./08-data-models.md)).
> 미해결 질문(Phase 2 후보): attribution postback 명세, 스크린샷 폴백 어뷰징, 파트너 온보딩 기준.

---

## 6.1 인사이트 ("노력 검증 불가" 문제도 해결)

우리는 *노력* 을 검증하지 않는다. 제휴 서비스로 라우팅하고 *인게이지먼트(설치/가입)* 를 검증한다 — 이건 체크 가능하고(어필리에이트 어트리뷰션) 우리에게 돈을 주는 그 행위다. 퀘스트는 저커밋 **행동**이지 **보고**가 아니다.

## 6.2 퀘스트 → 서비스 맵

| 퀘스트 도메인 | 파트너 타입 | 매출 이벤트 |
|---|---|---|
| 어학(TOEIC/OPIc) | 어학 인강 앱 | 설치 / 구독 |
| 자격증 | 자격증 플랫폼 | 강좌 구매(종종 자기개발비 지원) |
| 금융/적금 | 은행, 증권사 | 계좌 개설(CPA) |
| 학습/학점 | 강좌 플랫폼 | 수강 등록 |
| 취업(D-90) | 커리어 플랫폼 | 가입 |

## 6.3 2단계 완료

- **Tier 1 — 명예 탭 (기본, 마찰 0):** "30분 공부" → 한 탭 "완료." 퀘스트 포인트 + 스탯 XP. 항상 가능; 아무도 페이월 안 됨.
- **Tier 2 — Verified (선택, 더 큰 보상):** 우리 링크로 서비스 도달 → 어필리에이트 **postback** 이 퀘스트를 자동 ✓ verified 처리 → 보너스 XP + **verified 배지**(미래 Receipt로 이어짐). 어트리뷰션 없는 곳은 스크린샷 폴백.
- **넛지:** verified 가 보상↑ + 진짜 신뢰성을 주므로, 강압 없이 사용자가 스스로 매출 경로를 선택.

## 6.4 경제학

**퀘스트를 확정하는 어트리뷰션 postback = 파트너에게 청구하는 그 이벤트** — 한 파이프, 두 출력(검증 + 청구). 모델: CPI / CPA / 구독 rev-share. 매출은 인원수가 아니라 인게이지먼트에 스케일 — 시즌 코호트 비즈니스에 fit.

## 6.5 윤리 가드레일 (이 코호트는 레이더가 예리하다)

1. 진짜 도움 되는 서비스만 리퍼.
2. **정부 돈 먼저**(자기개발비 / 무료검정) — 병사가 자기 돈이 아니라 군대 돈을 쓰게.
3. 투명한 제휴 링크 라벨링.
4. **핵심 약속은 무료 유지** — 리퍼럴을 한 번도 안 탭한 병사도 자기 시간의 완전한 증명을 받는다.
